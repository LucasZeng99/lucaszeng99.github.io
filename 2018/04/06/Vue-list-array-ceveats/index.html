    • Problem

    when changing the values of an object or list, Vue cannot detect the change. For example:

    and in the data() function, pixels list is made of many small grid elements, say, “a pixel”, whose value is the color. I want to assign the color from my pen to the hovered blocks. With that in mind, I would bind the style to the color of a particular item.

    But that didn’t work out. the colors update only after I choose another pen color.

    Solution

    use this.$set(myArray, key, itemVal) instead of this.myArray[key] = item

    • Great thanks to James Thompson: Vuejs Forum
    • Vue.js official documentation: List Rendering

    What happened?

    • When changin a value in a Vue instance, it cannot detect it reactively.

    I find it hard for me to read into the internal code of Vue.js directly for now, and for this topic, I cannot say anything more than the official documentation: Change Detection Ceveats

    Vue cannot detect list/object change directly called, unless using its internal methods set to set the values and then triggers update.

    This is a workaround to javascript’s limited features, since when the value of list and object’s children is changed, the value of the address of the parent object/array will not, that result in a failed === judgement somewhere in Vue’s internal, where exactly I don’t know and I have zero confidence if I am right about this.
